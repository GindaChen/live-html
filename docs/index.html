live-html · v0.1

the rendered page is the source.

A tiny JS+CSS library that turns any single-file HTML page into a WYSIWYG editor and writes your edits back to disk. No build step, no framework, no database. Just an HTML file that knows how to edit itself.

the gap it fills

every slide stack splits source-edit and rendered-view. nobody round-trips.

the alternatives

slidev / reveal / marp / spectacle / quarto

All of them are one-way pipelines: write Markdown or JSX, build, render. To fix a typo you spotted on the rendered slide you flip back to the editor and find the line. Hot reload makes this fast — it does not make it round-trip.

the notion-shaped hole

round-trip editors lose the file

Notion, Coda, and friends round-trip just fine — but the artifact is a database row, not a git-friendly file you can commit, diff, grep, or hand to a coauthor.

what live-html does

the rendered DOM is the source of truth

Edits in the browser are captured into a coalesced op log, replayed against the in-memory document, and written back to the original .html file via the File System Access API. The artifact stays a single file. git diff shows what you changed.

why this works

HTML was always editable

The browser has had contenteditable for two decades and FSA writes since 2020. Nothing here is novel. It is a thin glue layer that nobody bothered to ship because everyone was busy building static-site generators.

architecture in 60 seconds

three layers, ~one file, no dependencies.

layer 1

contenteditable

Toggling edit mode flips contenteditable="true" on every matching node. Native selection, IME, paste — untouched. We add a focus outline and that's it.

layer 2

op log

Each input is captured as {path, prev, next, ts} and coalesced into the previous op if it's the same node within ~600ms. Persisted to localStorage so a refresh keeps your work. Undo walks the log, not the browser stack.

layer 3

File System Access

Save replays the log against documentElement.outerHTML, asks for a writable handle on the original file (cached after first grant), and writes. Safari/Firefox fall back to a download.

special-case

theme ops bake

CSS-variable edits don't mutate inline style — they collect into a single <style id="theme-overrides"> block. One block, one diff, regardless of how many tokens you touched.
